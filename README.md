1. В текущих версиях программы для Trace расчета берется одна Rocking curve для все
длин волн, что в корне неверно с физической точки зрения (на рис.1 представлен пример,
где видно, что Rocking curve отлична для разных длин волн).
Вариант 1. Нам хотелось бы, чтобы была возможность подгружать файл в виде 3
столбцов – wavelength, degrees и reflectivity (по сути 3D поверхность как на рис.1.) и далее
при «трейсе» программа брала нужную Rocking curve для каждой длины волны (тестовый
файл «Rocking curve test» с набором столбцов прикладываю к письму – открыть
можно с помощью ворда – столбец 1 – энергия в Эв; столбец 2 – отклонение от угла
брэгга; столбец 3 reflectivity).
Важно: в таблице значения wavelength будут заданы с некоторым шагом (например
0.5нм, 0.6нм, 07нм….), но может получиться так, что нам нужно значение длины волны,
которое не попадает в шаг (например 0.53нм), тогда программа должна считать значение
Rocking curve для ближайшей длины волны (в данном случае для 0.5нм, а не 0.6 нм).
**
Добавлена поддержка множества rocking curve.
Крывае задаются через файл с раширением *.curve как и ранее, но в стируктуре файла *.curve есть отличия.
Первая строка содержит в себе запись #v2 которая служит маркером нового формата кривой отражения. 
Далее идет строки задающие поверхность. В каждой строке записаны три значения:
1 – энергия в Эв; столбец 2 – отклонение от угла брэгга в градусах; столбец 3 reflectivity -- значение от 0 до 1.

пример:
```
#v2
       1675.0000   -0.0500000    0.0013507170
       1688.7895   -0.0500000    0.0012489737
       1702.5789   -0.0500000    0.0011576986
       1716.3684   -0.0500000    0.0010744043
       1730.1579   -0.0500000   0.00099803403
       1743.9474   -0.0500000   0.00092652673
	   .....
```
